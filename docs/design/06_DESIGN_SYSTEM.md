Version: 1.1

Status: Approved

# 1. Purpose

Dokumen ini mendefinisikan identitas visual resmi GHEVERHAN Commerce Platform.

Design System memastikan seluruh halaman, komponen, dan fitur memiliki tampilan yang konsisten.

Dokumen ini tidak mendefinisikan nilai teknis seperti ukuran, warna HEX, spacing, atau typography scale. Seluruh nilai teknis berada pada 07_DESIGN_TOKENS.md.

---

# 2. Design Vision

GHEVERHAN dibangun sebagai platform commerce modern yang mengutamakan:

- kesederhanaan
- konsistensi
- profesionalisme
- kecepatan
- kepercayaan

Setiap keputusan visual harus mendukung tujuan tersebut.

---

# 3. Brand Identity

Karakter visual GHEVERHAN adalah:

- Modern
- Premium
- Minimalist
- Elegant
- Professional
- Clean
- Timeless

Hindari gaya visual yang:

- ramai
- berlebihan
- terlalu banyak warna
- terlalu banyak dekorasi

---

# 4. Visual Language

Visual GHEVERHAN harus:

- mudah dikenali
- mudah dipahami
- fokus pada produk
- fokus pada konten
- tidak mengalihkan perhatian pengguna

UI mendukung produk, bukan menjadi pusat perhatian.

---

# 5. Color Philosophy

Palet warna digunakan untuk:

- membangun identitas
- memperjelas hirarki
- meningkatkan keterbacaan
- memberikan umpan balik

Seluruh definisi warna terdapat pada:

07_DESIGN_TOKENS.md

---

# 6. Typography Philosophy

Tipografi harus:

- mudah dibaca
- konsisten
- modern
- profesional

Gunakan maksimal dua keluarga font.

Pengaturan ukuran dan skala dijelaskan pada:

07_DESIGN_TOKENS.md

---

# 7. Iconography

Ikon digunakan untuk:

- memperjelas aksi
- mempercepat pemahaman
- mengurangi teks

Ikon tidak boleh digunakan hanya sebagai dekorasi.

Library ikon resmi:

Lucide Icons

---

# 8. Imagery

Seluruh gambar harus:

- berkualitas tinggi
- relevan
- konsisten
- fokus pada produk

Prioritaskan foto asli dibanding ilustrasi.

---

# 9. Layout Principles

Layout harus:

- memiliki hirarki jelas
- memberikan ruang kosong yang cukup
- mudah dipindai
- konsisten antar halaman

---

# 10. Interaction Principles

Interaksi harus memberikan umpan balik yang jelas.

Contoh:

Hover

↓

Active

↓

Loading

↓

Success

↓

Error

Pengguna tidak boleh menebak apakah aksi berhasil.

---

# 11. Motion Principles

Animasi digunakan hanya untuk:

- transisi
- feedback
- fokus

Animasi tidak boleh mengganggu pekerjaan pengguna.

---

# 12. Responsive Principles

Seluruh halaman harus mendukung:

- Mobile
- Tablet
- Desktop

Desain selalu dimulai dari Mobile.

---

# 13. Website Builder Guidelines

Website Builder harus tetap menjaga identitas visual GHEVERHAN.

Admin dapat mengubah:

- layout
- banner
- section
- warna tema
- konten

Namun perubahan tersebut tetap harus mengikuti Design System.

Website Builder tidak boleh menghasilkan tampilan yang tidak konsisten.

---

# 14. CMS Guidelines

Konten yang dikelola melalui CMS harus tetap mengikuti:

- tipografi
- layout
- spacing
- hierarchy

yang telah ditentukan oleh Design System.

---

# 15. Design Consistency Rules

Seluruh halaman wajib:

- menggunakan komponen resmi
- mengikuti Design Tokens
- mengikuti Component Library

Tidak diperbolehkan membuat variasi komponen tanpa alasan yang jelas.

---

# 16. Acceptance Criteria

□ Identitas visual konsisten.

□ Website Builder tetap menghasilkan tampilan yang konsisten.

□ CMS mengikuti Design System.

□ UI mudah dikenali sebagai GHEVERHAN.

□ Tidak ada elemen visual yang bertentangan.
