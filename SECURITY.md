# Security Policy

## Purpose

This document describes how security vulnerabilities should be reported for the GHEVERHAN Commerce Platform.

Security is a top priority for this project.

---

# Supported Versions

| Version | Supported |
|----------|-----------|
| v1.x | ✅ |
| Older Versions | ❌ |

Only the latest stable version receives security updates.

---

# Reporting a Vulnerability

If you discover a security vulnerability:

- Do NOT create a public GitHub Issue.
- Do NOT disclose the vulnerability publicly.
- Report it privately to the project maintainer.

Include:

- Description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Possible impact
- Suggested fix (optional)

---

# Response Process

Every report will follow this workflow.

Report

↓

Verification

↓

Risk Assessment

↓

Fix

↓

Testing

↓

Release

↓

Public Disclosure (if applicable)

---

# Scope

Security reports include:

- Authentication
- Authorization
- Payment
- API
- Session
- File Upload
- SQL Injection
- XSS
- CSRF
- Remote Code Execution
- Sensitive Data Exposure

---

# Out of Scope

The following are generally not considered security issues:

- UI bugs
- Typographical errors
- Performance issues
- Browser compatibility issues

Unless they introduce a security impact.

---

# Security Principles

GHEVERHAN follows:

- Least Privilege
- Secure by Default
- Defense in Depth
- Zero Trust
- Responsible Disclosure
